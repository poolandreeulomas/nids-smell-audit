
Guidelines for AI coding agents generating code in this repository.

## General Principles

- Prefer simple and minimal implementations
- Avoid unnecessary abstractions
- Focus on readable and maintainable code
- Do not introduce complex architectures unless explicitly requested

---

## Code Style

- Use clear variable and function names
- Keep functions small and focused
- Avoid deeply nested logic
- Prefer explicit code over clever tricks

---

## Project Simplicity

- Default to minimal working versions
- Avoid overengineering
- Do not add unnecessary dependencies

---

## Python Guidelines

- Follow PEP8 style
- Use type hints when helpful
- Keep modules small
- Write simple, clear functions

---

## Structure

- Keep files organized logically
- Separate API, logic, and UI when appropriate
- Avoid very large files

---

## Comments

- Add comments when the logic is not obvious
- Do not comment trivial lines
- Prefer short explanations over long paragraphs
